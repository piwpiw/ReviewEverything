# 전역 페이지 상세 설계 및 UI/UX 스펙 명세서 (PAGE_SPECS)

본 문서는 프로젝트(ReviewEverything) 내 **모든 프론트엔드 페이지의 세부 설계, UI/UX 철학, 데이터 바인딩 로직**을 유지 관리하고 업데이트하기 위한 기준 문서입니다. 
추후 AI Agent나 팀원이 새로운 페이지를 생성할 때, 이 문서에 명시된 **"Premium UI & 100x Quality"** 기준을 100% 동일하게 적용하여 단번에 최고 수준의 결과물을 뽑아내야 합니다.

---

## 🎨 1. 전역 Premium UI/UX 기준 (Global Standards)

새로운 컴포넌트나 페이지를 깎을 때는 무조건 다음의 요건을 충족해야 합니다.

1. **컬러 & 다크모드 (Tailwind)**
   - 백그라운드는 `bg-slate-50 dark:bg-slate-950` 혹은 `bg-white dark:bg-slate-900` 등 깊이감 있는 계층을 사용합니다.
   - 단색(Red, Blue) 대신 테마에 맞는 색상 (예: `blue-600`, `rose-500`, `amber-500`, `emerald-500`)과 투명도(opacity `10`, `20` 등)를 조합하여 글래스모피즘(Glassmorphism) 효과를 냅니다.
2. **타이포그래피 및 레이아웃**
   - 폰트는 시스템 폰트(Pretendard 등)를 상정하며, 제목은 `font-black tracking-tighter` 등 강렬한 굵기와 자간을 사용합니다.
   - 라벨이나 태그는 `text-[10px] font-black uppercase tracking-widest` 스타일을 사용하여 메타데이터의 시인성을 높입니다.
   - 메인 레이아웃은 `max-w-5xl` 또는 `max-w-[1400px]` 등의 컨테이너 안에서 좌측 리스트/그리드 뷰와 우측 사이드 패널(Meta/Detail) 구조로 분할합니다.
3. **인터랙션 및 애니메이션**
   - **Framer Motion** (`<motion.div>`)을 기본으로 사용하여 컴포넌트의 Mount/Unmount (`AnimatePresence`)와 리스트 렌더링 시 스르륵 나타나는 효과를 부여합니다.
   - 호버 이펙트(`group-hover`, `scale-95` on active)를 필수적으로 적용해 '살아있는 앱' 느낌을 줍니다.
4. **아이콘 및 데코레이션**
   - 모든 아이콘은 `lucide-react`를 사용합니다.
   - 빈 상태(Empty State) 디자인: 데이터가 없을 때는 텍스트만 덩그러니 두지 않고, 연관된 큰 아이콘(opacity-50 등)과 부드러운 가이드라인 텍스트, '추가/재시도' 버튼 패턴을 묶어서 배치합니다.
5. **데이터 바인딩 및 Fallback (모의 데이터)**
   - 로컬 테스트나 DB 에러 시 화면이 깨지지 않도록 모든 API 연동부 렌더링에 **고도화된 Mock Fallback 로직**을 탑재하여 언제든 UI 컴포넌트가 온전히 렌더링되도록 보장합니다.

## 📄 2. 페이지별 상세 스펙 (Page Specifications)

해당 이력을 바탕으로 모든 페이지를 수정/생성합니다.

### 2.1 스마트 캘린더 (`/me/calendar`)
**목적:** 사용자의 캠페인 진행 상태를 6가지로 세분화하여 현미경 단위로 추적하고 외부 알람을 통합하는 개인화 페이지.

* **상태 세분화 (Status Config)**
  - `APPLIED` (선발대기) : `slate` 기반 회색조 컬러
  - `SELECTED` (예약대기) : `violet` 보라색조 컬러
  - `SCHEDULED` (방문예정) : `blue` 메인 컬러
  - `REVIEWING` (리뷰대기) : `amber` 경고/대기 컬러
  - `URGENT` (마감임박) : `rose` 붉은색조 컬러 (D-2 이내)
  - `COMPLETED` (등록완료) : `emerald` 성공 컬러
* **레이아웃 구조**
  - **헤더:** 좌측 대시보드 복귀 링크, 중앙 제목, 우측 현재 월 제어 컨트롤 및 '새 일정' 버튼
  - **좌측 (월간 그리드):** 7열 그리드 (일~토), 호버 시 `hover:z-10` 확대, 날짜 선택 시 파란색 링 강조(`ring-2`)
  - **우측 (사이드 패널):**
    - **외부 알람 연동 (구독):** 카카오톡 봇 알림, Google 캘린더, 이메일 리포트 각각 토글 형태. 클릭 시 UI가 React State 기반으로 On/Off 되며 시각적 피드백 제공.
    - **디테일 뷰:** 날짜 클릭 시 해당 일자의 6가지 상태 라벨이 달린 스케줄 리스트 노출. 일정이 없는 경우 세련된 빈 화면 일러스트/아이콘 처리. 커스텀 추가 버튼 연결.
* **Fallback 전략**
  - `/api/me/board` 호출 실패 시 이번 달 기준으로 8개의 완성도 높은 가상 시나리오(신청, 방문, 마감 등) 데이터가 바인딩되도록 작성.

### 2.2 메인 탐색 & Curation (`/` & `app/page.tsx`)
**목적:** 사용자가 최고 효율의 캠페인을 1초 만에 찾을 수 있게 하는 포털 경험 제공.

* **히어로(Hero) 섹션**
  - 필터가 비어있을 때만 거대한 배너(Moaview Phase 2 Hero Hub)가 나타나며, 백그라운드에 `blur-[100px]` 먹인 동적 그라데이션 원형 도형 배치.
  - 검색어 입력(`HeroSearch`), 실시간 수집 지표(`StatsBanner`)가 중앙에 떠 있음.
* **상단 필터 바 (`FilterBar`)**
  - 스크롤 시 상단에 고정(`sticky top-[64px] z-50 backdrop-blur-xl`).
* **데이터 리스트 뷰 & 맵 뷰 분기**
  - `view=list`와 `view=map` 쿼리 파라미터에 따라 자연스럽게 컴포넌트를 교체.
  - `LayoutGrid`, `MapIcon` 탭은 그림자와 배경색으로 현재 활성 상태 직관적 표시.

### 2.3 관리자 커맨드 데스크 (`/admin`)
**목적:** 시스템 인프라 수집과 장애를 컨트롤하는 터미널 감성의 초고급 제어 장치.

* **HUD 컨셉**
  - 검정색 배경(`bg-[#020617]`) 기반의 사이버네틱 UI 적용.
  - 4개의 시스템 지표(DB 연결, 런 성공, 시스템 오류, 타임스탬프)가 상단에 떠 있음.
* **IO Stream Terminal**
  - 실시간 로그를 담는 검정 박스와 녹색 텍스트(`text-emerald-500 font-mono`) 콘솔 구현.
  - `[INITIATE_INGESTION]` 버튼에 글래스모피즘 그라데이션 적용, 버튼 클릭 시 터미널창 리로드.
* **Fallback 전략**
  - Auth: 로컬 환경(`NODE_ENV !== 'production'`)일 때 `ADMIN_PASSWORD` 방어 로직 우회하여 에러 발생 원천 차단.
  - API: DB 오프라인 시 500이 아닌 200 OK + 로컬 목업 데이터(5대 플랫폼 가상 에러 로그 포함) 응답.

## 🛠️ 3. 페이지 확장 가이드라인 (에이전트 주의사항)

- **문서 동기화 보장:** 새로운 페이지 생성/업데이트를 마친 직후에는 **반드시 본 문서(`docs/PAGE_SPECS.md`)에 상세 설계를 덧붙여서 업데이트** 해야 합니다.
- **"동일 수준" 유지:** 캘린더 페이지(`me/calendar`)나 관리자 페이지(`admin`)의 코드는 일종의 컴포넌트 조립 기준점입니다. 새로운 폼(Form), 차트(Chart), 리스트(List)를 만들 때 기존에 사용한 Tailwind 클래스와 레이아웃/아이콘 패턴을 복사하여 일관성을 맞추십시오.
- **백데이터 필수 장착:** 새로 기획한 화면은 사용자가 직관적으로 이해할 수 있어야 합니다. 모든 화면은 로컬 테스트가 즉시 가능하게끔 `component` 또는 `page`나 `/api/` 내부에 `catch` 블록을 활용하여 **완성된 스토리보드를 담은 가상 배열(Mock Array)** 을 반드시 리턴하십시오.
